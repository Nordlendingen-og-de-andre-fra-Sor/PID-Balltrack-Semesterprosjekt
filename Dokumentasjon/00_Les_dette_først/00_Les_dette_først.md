# Les dette først

Denne dokumentasjonen utdyper prosjektet beskrevet i README.md.

Formålet med dokumentasjonen er å:
- forklare hvordan systemet er bygget opp
- beskrive hvordan koden er strukturert
- dokumentere hardwareoppsett og reguleringsstruktur
- gjøre prosjektet lett å overta og videreutvikle

Dokumentasjonen er ment å leses i følgende rekkefølge:
1. Systemoversikt
2. Hardwareoppsett
3. Reguleringsstruktur og PID
4. Kode og GUI
5. Raspberry Pi og terminal
